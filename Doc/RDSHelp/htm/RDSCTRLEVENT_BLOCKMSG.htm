Б.2.2. RDSCTRLEVENT_BLOCKMSG – сообщение от блока

Событие RDSCTRLEVENT_BLOCKMSG наступает при вызове моделью одного из блоков схемы функции rdsRemoteControllerCall – с ее помощью блок может по собственной инициативе передать внешней управляющей программе произвольную строку и целое число. Реакция на это событие возможна следующими способами:

  • с помощью стандартной функции обратного вызова, зарегистрированной вызовом rdsctrlRegisterEventStdCallback, при этом имя передавшего сообщение блока, переданное число и строка передаются в третьем параметре этой функции через структуру RDSCTRL_BLOCKMSGDATA;
  • с помощью сообщения окну управляющей программы, зарегистрированному функцией rdsctrlRegisterEventMessage, при этом имя передавшего сообщение блока, переданное число и строка передаются в функцию через структуру RDSCTRL_BLOCKMSGDATA, указатель на которую находится в поле Data структуры RDSCTRL_MSGEVENTDATA, указатель на которую, в свою очередь, передается в параметре LPARAM оконного сообщения;
  • с помощью специализированной функции обратного вызова, зарегистрированной вызовом rdsctrlRegisterBlockMsgCallback, при этом имя передавшего сообщение блока, переданное число и строка передаются в параметрах функции.

Пример

Пример реакции на событие RDSCTRLEVENT_BLOCKMSG приведен в §3.4 руководства программиста.

См. также

Способы реакции на события, rdsRemoteControllerCall, rdsctrlRegisterEventStdCallback, RDSCTRL_BLOCKMSGDATA, rdsctrlRegisterEventMessage, RDSCTRL_MSGEVENTDATA, rdsctrlRegisterBlockMsgCallback.
