Б.1.4. RDSCTRL_NEWFILEDATA – описание события смены загруженной схемы

В структуре RDSCTRL_NEWFILEDATA передаются параметры сообщения RDS о смене текущей рабочей схемы RDSCTRLEVENT_NEWFILE.

  typedef struct {
    DWORD servSize;    // Размер этой структуры
    int Reason;        // Причина события (RDSCTRLEVENT_NEWFILE_*)
    LPSTR FileName;    // Имя файла
  } RDSCTRL_NEWFILEDATA;
  typedef RDSCTRL_NEWFILEDATA *RDSCTRL_PNEWFILEDATA;

Поля структуры

servSize (DWORD)
Размер этой структуры в байтах. В это поле автоматически заносится значение sizeof(RDSCTRL_NEWFILEDATA). Управляющая программа может использовать это поле для проверки соответствия размера переданной структуры собственным ожиданиям.
Reason (int)
Одна из констант RDSCTRLEVENT_NEWFILE_*, указывающая на причину смены текущего файла схемы, с которым работает управляемая копия RDS:
RDSCTRLEVENT_NEWFILE_NEW Создана новая пустая схема.
RDSCTRLEVENT_NEWFILE_TEMPLATE Новая схема создана по файлу шаблона.
RDSCTRLEVENT_NEWFILE_LOAD Загружен файл схемы.
FileName (LPSTR)
Указатель на строку с именем загруженного файла схемы или файла шаблона, по которому создана новая схема. Если событие произошло из-за создания новой пустой схемы, в этом поле записан указатель на пустую строку.

Примечания

Указатель на эту структуру передается управляющей программе двумя способами:

  • Если для реакции используется функция обратного вызова, зарегистрированная при помощи функции rdsctrlRegisterEventStdCallback, то указатель на RDSCTRL_NEWFILEDATA передается в третьем параметре функции обратного вызова.
  • Если для реакции используется сообщение окну управляющей программы, зарегистрированному функцией rdsctrlRegisterEventMessage, то указатель на RDSCTRL_NEWFILEDATA передается в поле Data структуры RDSCTRL_MSGEVENTDATA, указатель на которую, в свою очередь, передается в параметре LPARAM оконного сообщения.

Структура RDSCTRL_NEWFILEDATA и строка, на которую указывает ее поле FileName, существуют только во время выполнения реакции управляющего приложения на событие, то есть до завершения функции обратного вызова или реакции на оконное сообщение.
