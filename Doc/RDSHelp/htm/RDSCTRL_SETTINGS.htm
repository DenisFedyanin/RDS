Б.1.7. RDSCTRL_SETTINGS – общие параметры RDS

Структура RDSCTRL_SETTINGS используется для описания общих параметров RDS при их получении функцией rdsctrlGetGeneralSettings и установке функцией rdsctrlSetGeneralSettings.

  typedef struct {
    DWORD servSize;         // Размер этой структуры
    DWORD UIFlags;          // Настройки пользовательского
                            // интерфейса (RDSCTRL_SET_*)
    BOOL SystemAutoSave;    // Автосохранение при выходе
  } RDSCTRL_SETTINGS;
  typedef RDSCTRL_SETTINGS *RDSCTRL_PSETTINGS;

Поля структуры

servSize (DWORD)
Размер этой структуры в байтах. Перед вызовом любой из функций, работающих со структурой, управляющая программа должна записать в это поле значение sizeof(RDSCTRL_SETTINGS).
UIFlags (DWORD)
Набор битовых флагов, указывающих на разрешение или запрещение элементов пользовательского интерфейса RDS (флаги объединяются битовым ИЛИ):
RDSCTRL_SET_ENABLECALC Разрешено включение режима моделирования, см. также rdsctrlEnableCalcMode.
RDSCTRL_SET_ENABLEEDIT Разрешено включение режима редактирования, см. также rdsctrlEnableEditMode.
RDSCTRL_SET_ENABLEMAINWIN Включена видимость главного окна RDS, см. также rdsctrlShowMainWindow.
RDSCTRL_SET_ENABLEOPTIONS Разрешено изменение настроек RDS, см. также rdsctrlEnableOptions.
RDSCTRL_SET_ENABLEPARAMED Разрешено изменение основных параметров блоков – управляющей функции, внешнего вида и т.п., см. также rdsctrlEnablePropEdit.
RDSCTRL_SET_ENABLERUN Разрешено включение режима расчета, см. также rdsctrlEnableRun.
RDSCTRL_SET_ENABLESYSWIN Разрешено открытие окон подсистем, см. также rdsctrlEnableSubsystemWindows.
RDSCTRL_SET_ENABLEUI Разрешен пользовательский интерфейс RDS, см. также rdsctrlEnableUI.
Различные флаги управляют различными функциями интерфейса: при установленном флаге функция разрешена, при сброшенном – запрещена. Битовый флаг RDSCTRL_SET_ENABLEUI управляет всем интерфейсом в целом: если он сброшен, работа RDS будет невидима для пользователя.
SystemAutoSave (BOOL)
TRUE – при выходе из RDS или загрузке новой схемы автоматически сохранять текущую, если в ней были изменения. FALSE – обычное поведение: автоматическое сохранение не производится. Для управления автоматическим сохранением можно также использовать функцию rdsctrlSetAutoSave.

Примечания

Разрешать и запрещать отдельные элементы пользовательского интерфейса можно специализированными сервисными функциями, описанными в приложении Б.3.3.

См. также

rdsctrlGetGeneralSettings, rdsctrlSetGeneralSettings, rdsctrlSetAutoSave.
