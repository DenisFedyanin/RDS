А.5.32.7. Команда RDS_CSV_FILEERROR – проверка ошибки в последней операции с файлом

Команда RDS_CSV_FILEERROR позволяет проверить, успешно ли завершилась последняя операция объекта с файлом.

Вызов команды

  int iError=rdsGetObjectInt(Csv,RDS_CSV_FILEERROR,0);

Параметр и результат

Csv (RDS_HOBJECT)
Идентификатор вспомогательного объекта для работы с CSV, ранее созданного функцией rdsCSVCreate.
iError (int)
Нулевое значение – ошибок не было, отличное от нуля значение – последняя операция выполнена с ошибкой.

Примечания

Эта команда используется для проверки результата выполнения команд работы с файлом: RDS_CSV_STRFROMFILE, RDS_CSV_STRTOFILE, RDS_CSV_OPENFILEREAD, RDS_CSV_OPENFILEWRITE, RDS_CSV_LOADFROMFILE и RDS_CSV_SAVETOFILE. Не во всех этих командах предусмотрен возврат результата, поэтому для того, чтобы узнать, успешно ли выполнена такая команда, после нее следует вызвать RDS_CSV_FILEERROR.

См. также

rdsCSVCreate, rdsSetObjectInt, rdsGetObjectInt, RDS_CSV_FILEISOPEN.
