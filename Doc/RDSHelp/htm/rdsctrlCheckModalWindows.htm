Б.3.4.8. rdsctrlCheckModalWindows – проверить наличие модальных окон

Функция rdsctrlCheckModalWindows проверяет наличие открытых модальных окон в управляемой копии RDS.

  BOOL RDSCALL rdsctrlCheckModalWindows(
    int Link    // Идентификатор связи
  );

Тип указателя на эту функцию

RDSCTRL_BI

Параметр

Link (int)
Идентификатор связи с RDS, присвоенный ей при создании функцией rdsctrlCreateLink.

Возвращаемое значение

TRUE – есть открытые модальные окна, FALSE – таких окон нет.

Примечания

Эта функция возвращает TRUE, если в копии RDS, управляемой через связь Link, в данный момент есть открытые модальные окна. Модальные окна могут помешать некоторым действиям (при открытом модальном окне нежелательно пытаться открыть немодальное – например, окно подсистемы), поэтому управляющей программе может понадобиться эта информация. Для принудительного закрытия всех открытых в данный момент модальных окон можно использовать функцию rdsctrlCloseModalWindows.

RDS автоматически отслеживает собственные модальные окна и окна, открываемые с помощью сервисных функций. Если модель блока или модуль автоматической компиляции открывает модальные окна при помощи функций Windows API или сторонних библиотек, необходимо информировать об этом RDS, вызывая перед открытием такого модального окна функцию rdsBlockModalWinOpen, а после его закрытия – rdsBlockModalWinClose.

См. также

rdsctrlCreateLink, rdsctrlCloseModalWindows, rdsBlockModalWinOpen, rdsBlockModalWinClose.
