Б.3.2.4. rdsctrlDeleteLink – уничтожить связь с RDS

Функция rdsctrlDeleteLink уничтожает связь, через которую происходит управление запущенной копией RDS.

  void RDSCALL rdsctrlDeleteLink(
    int Link    // Идентификатор связи
  );

Тип указателя на эту функцию

RDSCTRL_VI

Параметр

Link (int)
Идентификатор связи с RDS, присвоенный ей при создании функцией rdsctrlCreateLink.

Примечания

Эта функция уничтожает связь с идентификатором Link, больше эту связь нельзя будет использовать для запуска RDS и обмена данными с блоками схемы. Если на момент уничтожения связи через нее управлялась копия RDS, эта копия автоматически завершается.

Идентификаторы связей могут использоваться повторно, поэтому при очередном вызове rdsctrlCreateLink библиотека RdsCtrl.dll может присвоить новой созданной связи идентификатор одной из уничтоженных ранее.

Для завершения RDS не обязательно уничтожать связь – вместо этого можно вызвать функции rdsctrlDisconnect или rdsctrlClose и использовать связь с идентификатором Link позднее для запуска новой копии RDS с теми же параметрами и настройками.

Пример

Пример использования функции rdsctrlDeleteLink приведен в §3.2 руководства программиста.

См. также

rdsctrlCreateLink, rdsctrlDisconnect, rdsctrlClose.
