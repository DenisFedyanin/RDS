Б.3.3.7. rdsctrlEnableSubsystemWindows – разрешение открытия окон подсистем

Функция rdsctrlEnableSubsystemWindows разрешает или запрещает открытие окон подсистем, включая окно корневой подсистемы.

  void RDSCALL rdsctrlEnableSubsystemWindows(
    int Link,      // Идентификатор связи
    BOOL Enable    // Разрешение
  );

Тип указателя на эту функцию

RDSCTRL_VIB

Параметры

Link (int)
Идентификатор связи с RDS, присвоенный ей при создании функцией rdsctrlCreateLink.
Enable (BOOL)
TRUE – иоткрытие окон подсистем разрешено, FALSE – запрещено.

Примечания

Эта функция разрешает (при Enable==TRUE) или запрещает (при Enable==FALSE) как ручное, так и программное открытие окон любых подсистем в копии RDS, управляемой через связь Link (по умолчанию открытие окон разрешено). Чаще всего она используется при работе с портами вывода, когда содержимое подсистем отображается не в отдельных окнах, принадлежащих RDS, а в собственном окне управляющего приложения.

Разрешение открытия окон запоминается в параметрах связи Link, его не нужно устанавливать заново при повторном запуске RDS через эту связь. Можно, например, вызвать rdsctrlEnableSubsystemWindows(Link,FALSE) сразу после создания связи, когда еще нет работающей копии RDS. При этом после вызова rdsctrlConnect RDS запустится уже с заблокированным открытием окон подсистем.

Пример

Пример использования функции rdsctrlEnableSubsystemWindows приведен в §3.6.1 руководства программиста.

rdsctrlCreateLink, rdsctrlSetGeneralSettings, rdsctrlConnect.
