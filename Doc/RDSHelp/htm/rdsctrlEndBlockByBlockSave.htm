Б.3.5.3. rdsctrlEndBlockByBlockSave – завершить поблочное сохранение схемы

Функция rdsctrlEndBlockByBlockSave завершает или отменяет поблочное сохранение схемы, начатое вызовом rdsctrlStartBlockByBlockSave.

  void RDSCALL rdsctrlEndBlockByBlockSave(
    int Link    // Идентификатор связи
  );

Тип указателя на эту функцию

RDSCTRL_VI

Параметр

Link (int)
Идентификатор связи с RDS, присвоенный ей при создании функцией rdsctrlCreateLink.

Примечания

Эта функция завершает поблочную передачу схемы из копии RDS, управляемой через связь Link, в управляющее приложение. Поблочное сохранение начинается вызовом rdsctrlStartBlockByBlockSave, за которым следуют вызовы rdsctrlGetBlockByBlockSavePiece, каждый их которых получает из RDS блок, связь, шину или какие-либо вспомогательные данные. После того, как данные всех блоков и связей получены из RDS, управляющее приложение вызывает rdsctrlEndBlockByBlockSave для освобождения вспомогательной памяти, отведенной «RdsCtrl.dll» для временного хранения передаваемых данных.

Пример

Пример использования функции rdsctrlEndBlockByBlockSave приведен в §3.5 руководства программиста.

См. также

rdsctrlCreateLink, rdsctrlGetBlockByBlockSavePiece, rdsctrlStartBlockByBlockSave.
