Б.3.7.3. rdsctrlGetVPMouseLevel – получить тип реакции на мышь у подсистемы в порте вывода

Функция rdsctrlGetVPMouseLevel возвращает тип реакции подсистемы, привязанной в данный момент к указанному порту вывода, на действия пользователя мышью.

  int RDSCALL rdsctrlGetVPMouseLevel(
    int Link,    // Идентификатор связи
    int VpId     // Идентификатор порта вывода
  );

Тип указателя на эту функцию

RDSCTRL_III

Параметры

Link (int)
Идентификатор связи с RDS, присвоенный ей при создании функцией rdsctrlCreateLink.
VpId (int)
Идентификатор порта вывода, присвоенный ему при создании функцией rdsctrlSetViewport.

Возвращаемое значение

Тип реакции подсистемы на мышь:

0 ни окно подсистемы, ни ее блоки не реагируют ни на какие действия мышью;
1 окно подсистемы или хотя бы один из ее блоков реагирует на нажатия кнопок и перемещение курсора с нажатыми кнопками, но никто в подсистеме не реагирует на перемещение курсора без нажатия кнопок;
2 окно подсистемы или один из ее блоков реагирует на перемещения курсора мыши без нажатия кнопок.

Примечания

Эта функция возвращает особенности реакции подсистемы, привязанной в данный момент к порту вывода VpId в копии RDS, управляемой через связь Link, на действия пользователя мышью. Обычно она используется для того, чтобы определить, какие именно события, связанные с мышью, управляющее приложение должно передавать в RDS (постоянная передача всех действий пользователя в RDS приводит к ненужной нагрузке на связь между управляющим приложением и RDS). Если функция вернула нулевое значение, никакую информацию о действиях мышью передавать в RDS не нужно: в подсистеме некому ее обрабатывать. Если функция вернула значение 1, в RDS следует передавать только нажатия и отпускания кнопок мыши и перемещения курсора при нажатых кнопках. Если же функция вернет значение 2, в RDS нужно передавать все действия пользователя.

Для передачи в RDS информации о движениях курсора мыши и нажатии и отпускании ее кнопок используется функция rdsctrlViewportMouse.
