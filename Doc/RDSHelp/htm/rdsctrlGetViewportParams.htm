Б.3.7. Функции для работы с портами вывода

Описываются функции, управляющие работой портов вывода (см. §3.6 руководства программиста), то есть рисованием подсистем схемы внутри окна управляющего приложения.

Б.3.7.1. rdsctrlGetViewportParams – получить масштаб и сдвиг подсистемы в порте вывода

Функция rdsctrlGetViewportParams возвращает масштаб и положение изображения подсистемы в указанном порте вывода.

  void RDSCALL rdsctrlGetViewportParams(
    int Link,         // Идентификатор связи
    int VpId,         // Идентификатор порта вывода
    double *pZoom,    // Возвращаемый масштаб
    int *pScrollX,    // Возвращаемый горизонтальный сдвиг
    int *pScrollY     // Возвращаемый вертикальный сдвиг
  );

Тип указателя на эту функцию

RDSCTRL_VIIpDpIpI

Параметры

Link (int)
Идентификатор связи с RDS, присвоенный ей при создании функцией rdsctrlCreateLink.
VpId (int)
Идентификатор порта вывода, присвоенный ему при создании функцией rdsctrlSetViewport.
pZoom (double*)
Указатель на вещественную переменную, в которую функция запишет текущий масштаб порта вывода в долях единицы (1.0 – 100%, 0.5 – 50% и т.д.). Если управляющей программе не нужен масштаб, в этом параметре можно передать NULL.
pScrollX, pScrollY (int*)
Указатели на целые переменные, в которые функция запишет горизонтальную (pScrollX) и вертикальную (pScrollY) координаты левого верхнего угла области рабочего поля подсистемы, видимой в порте вывода. Координаты возвращаются в точках экрана в текущем масштабе подсистемы. Если управляющей программе не нужны эти координаты, в этих параметрах можно передать NULL.

Примечания

Эта функция возвращает параметры отображения подсистемы, установленные для порта вывода VpId в копии RDS, управляемой через связь Link. Координаты левого верхнего угла видимой в порте области возвращаются в точках экрана в текущем установленном для порта масштабе, поэтому их можно использовать для отображения полос прокрутки.
