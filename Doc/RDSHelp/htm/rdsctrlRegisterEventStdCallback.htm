Б.3.6.4. rdsctrlRegisterEventStdCallback – регистрация функции для реакции на событие

Функция rdsctrlRegisterEventStdCallback регистрирует в «RdsCtrl.dll» функцию управляющего приложения, которая будет вызываться при при наступлении в RDS указанного события.

  void RDSCALL rdsctrlRegisterEventStdCallback(
    int Link,                  // Идентификатор связи
    int Event,                 // Идентификатор события (RDSCTRLEVENT_*)
    RDSCTRL_CALLBACK CallBack, // Функция
    LPVOID pAuxData            // Дополнительные данные
  );

Тип указателя на эту функцию

RDSCTRL_VIICbpV

Параметры

Link (int)
Идентификатор связи с RDS, присвоенный ей при создании функцией rdsctrlCreateLink.
Event (int)
Целый идентификатор события, реакция на которое регистрируется (см. приложение Б.2), то есть одна из констант RDSCTRLEVENT_*.
CallBack (RDSCTRL_CALLBACK)
Указатель на функцию, которая будет вызываться при каждом событии Event.
pAuxData (LPVOID)
Указатель на дополнительные данные, который будут передаваться в функцию CallBack при каждом вызове.

Примечания

Эта функция запоминает для связи Link указатель на функцию, которая будет вызываться каждый раз при наступлении события Event в копии RDS, управляемой через связь Link. Функция, указатель на которую передается в параметре CallBack, должна иметь следующий вид:

  void RDSCALL имя_функции(
    int Link,         // Идентификатор связи с RDS
    int Event,        // Идентификатор события
    LPVOID pEData,    // Данные события
    LPVOID pAux       // Дополнительные данные
  );

В параметре Link этой функции передается идентификатор связи с управляемой копией RDS, в которой произошло событие (можно зарегистрировать одну и ту же функцию для нескольких связей), в параметре Event – идентификатор наступившего события (можно зарегистрировать одну и ту же функцию для разных событий), в параметре pEData – указатель на структуру, описывающую событие (эта структура у каждого события своя), и в параметре pAux – указатель, переданный в параметре pAuxData функции rdsctrlRegisterEventStdCallback при регистрации реакции (это единственный способ передать в функцию реакции какие-либо дополнительные параметры).

Регистрация функции реакции на событие отменяет прежнюю регистрацию реакции на это же событие для связи Link, независимо от того, как именно эта реакция была зарегистрирована.

Пример

Пример использования функции rdsctrlRegisterEventStdCallback приведен в §3.4 руководства программиста.
