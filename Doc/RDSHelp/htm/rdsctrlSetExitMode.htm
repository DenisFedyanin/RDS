Б.3.3.11. rdsctrlSetExitMode – установить предупреждение о выходе из RDS

Функция rdsctrlSetExitMode устанавливает одно из двух возможных предупреждений пользователю о завершении RDS, либо отключает такое предупреждение.

  void RDSCALL rdsctrlSetExitMode(
    int Link,    // Идентификатор связи
    int Mode     // Вид предупреждения
  );

Тип указателя на эту функцию

RDSCTRL_VII

Параметры

Link (int)
Идентификатор связи с RDS, присвоенный ей при создании функцией rdsctrlCreateLink.
Mode (int)
Целое число, указывающее вид предупреждения пользователю:
0 Выход из RDS без предупреждения (по умолчанию).
1 Пользователю выводится предупреждение о том, что RDS в данный момент управляется через внешнее приложение, и он должен подтвердить завершение RDS или отказаться от него.
2 Пользователю выводится сообщение о том, что RDS управляется через внешнее приложение, выход из RDS запрещен.

Примечания

Эта функция управляет реакцией на попытку пользователя завершить RDS при внешнем управлении. По умолчанию RDS завершается без каких-либо сообщений пользователю. Чаще всего эту функцию используют для информирования пользователя о том, что завершение RDS в данный момент может повлиять на работу управляющей программы. Если пользовательский интерфейс RDS запрещен, использование этой функции не имеет смысла, поскольку пользователь не сможет завершить RDS по собственной инициативе.

Установленный тип сообщения пользователю запоминается в параметрах связи Link, его не нужно устанавливать заново при повторном запуске RDS через эту связь. Можно, например, вызвать rdsctrlSetExitMode сразу после создания связи, когда еще нет работающей копии RDS. При этом после вызова rdsctrlConnect RDS запустится уже с запомненным типом предупреждения о выходе.
