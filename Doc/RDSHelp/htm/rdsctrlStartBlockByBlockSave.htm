Б.3.5.18. rdsctrlStartBlockByBlockSave – начать поблочное сохранение схемы

Функция rdsctrlStartBlockByBlockSave начинает поблочную передачу схемы из управляемой копии RDS в управляющую программу, при которой каждый блок, связь или шина схемы считываются из RDS отдельным вызовом.

  int RDSCALL rdsctrlStartBlockByBlockSave(
    int Link,      // Идентификатор связи
    DWORD Flags    // Флаги (не используются)
);

Тип указателя на эту функцию

RDSCTRL_IIDw

Параметры

Link (int)
Идентификатор связи с RDS, присвоенный ей при создании функцией rdsctrlCreateLink.
Flags (DWORD)
Флаги функции (в настоящее время этот параметр никак не используется).

Возвращаемое значение

Общее число объектов, доступных для считывания.

Примечания

Эта функция начинает поблочную передачу схемы из копии RDS, управляемой через связь Link. После ее вызова можно последовательно получать из RDS описания различных объектов схемы функцией rdsctrlGetBlockByBlockSavePiece, для завершения передачи схемы следует вызвать rdsctrlEndBlockByBlockSave. Функция возвращает число объектов (блоков, связей, шин, дополнительных описаний), которые можно считать из RDS. Для получения всех объектов схемы можно либо вызвать rdsctrlGetBlockByBlockSavePiece указанное число раз, либо вызывать ее до тех пор, пока она не вернет константу RDS_SFTAG_EOF (признак конца объектов).

Функция rdsctrlStartBlockByBlockSave создает в памяти большой объем дополнительных структур, используемых при чтении объектов, поэтому крайне важно вызвать rdsctrlEndBlockByBlockSave как можно быстрее после завершения считывания схемы, чтобы освободить эту память.

Пример

Пример использования функции rdsctrlStartBlockByBlockSave приведен в §3.5 руководства программиста.
