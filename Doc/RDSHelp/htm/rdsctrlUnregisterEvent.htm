Б.3.6.5. rdsctrlUnregisterEvent – отмена регистрации реакции на событие

Функция rdsctrlUnregisterEvent прекращает реакцию управляющего приложения на указанное событие RDS.

  void RDSCALL rdsctrlUnregisterEvent(
    int Link,    // Идентификатор связи
    int Event    // Событие
  );

Тип указателя на эту функцию

RDSCTRL_VII

Параметры

Link (int)
Идентификатор связи с RDS, присвоенный ей при создании функцией rdsctrlCreateLink.
Event (int)
Целый идентификатор события, реакция на которое отменяется (см. приложение Б.2), то есть одна из констант RDSCTRLEVENT_*.

Примечания

После вызова этой функции управляющее приложение перестает получать из копии RDS, управляемой через связь Link, информацию о наступлении события Event, если ранее реакция на это событие была зарегистрирована. Регистрация реакции на событие отменяется независимо от того, как именно эта реакция осуществлялась: через функцию обратного вызова (rdsctrlRegisterBlockMsgCallback, rdsctrlRegisterEventStdCallback) или через оконное сообщение (rdsctrlRegisterEventMessage).

См. также

Способы реакции на события, rdsctrlRegisterEventMessage, rdsctrlRegisterBlockMsgCallback, rdsctrlRegisterEventStdCallback, rdsctrlCreateLink.
