Б.3.7.15. rdsctrlViewportSystem – получить имя подсистемы в порте вывода

Функция rdsctrlViewportSystem возвращает полное имя подсистемы, привязанной в данный момент к указанному порту вывода.

  BOOL RDSCALL rdsctrlViewportSystem(
    int Link,         // Идентификатор связи
    int VpId,         // Идентификатор порта вывода
    LPVOID SysName    // Возвращаемое имя подсистемы
  );

Тип указателя на эту функцию

RDSCTRL_BIIpV

Параметры

Link (int)
Идентификатор связи с RDS, присвоенный ей при создании функцией rdsctrlCreateLink.
VpId (int)
Идентификатор порта вывода, присвоенный ему при создании функцией rdsctrlSetViewport.
SysName (LPVOID)
Указатель на объект управляющего приложения, в который функция обратного вызова, зарегистрированная при помощи rdsctrlSetStringCallback, запишет строку с полным именем подсистемы. Для корневой подсистемы будет возвращена пустая строка.

Возвращаемое значение

TRUE – функция выполнена успешно, FALSE – ошибка (нет такого порта и т.п.).

Примечания

Эта функция записывает в объект, указатель на который передан через параметр SysName, полное имя подсистемы привязанной к порту вывода VpId в копии RDS, управляемую через связь Link. Для возврата строки используется функция обратного вызова, ранее зарегистрированная через rdsctrlSetStringCallback.

Возвращаемое полное имя подсистемы начинается с двоеточия, за которым следует последовательное перечисление через двоеточие всех имен подсистем на пути от корневой подсистемы до этой. Например, полное имя «:Sys1:Sys100:MySys» говорит о том, что подсистема с именем «MySys» находится в подсистеме «Sys100», которая, в свою очередь, находится в подсистеме «Sys1» корневой подсистемы.

Пример

Пример использования функции rdsctrlViewportSystem приведен в §3.6.3 руководства программиста.

rdsctrlCreateLink, rdsctrlSetViewport, rdsctrlSetStringCallback.
